# portal-ou-hub

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库本身不绑定任何特定域名或网站，仅作为 HTML 资源集合的存储与分发点。

## 项目简介

本仓库旨在提供一种结构化的方式，将分散的、独立的 HTML 页面统一管理。每个页面可以是一个工具、一个信息页、一个交互原型，或任何适合以 HTML 单页形式呈现的内容。仓库本身不提供导航或聚合功能，仅作为文件载体。

## 目录说明

```
.
├── pages/          # 存放所有独立 HTML 页面，每个文件为一个页面
├── assets/         # 全局共享资源（如 CSS、JS、图片），按需使用
├── docs/           # 相关文档或说明
└── README.md       # 本文件
```

- `pages/` 目录下每个 `.html` 文件即为一个独立页面，文件名即为该页面的标识。
- `assets/` 目录用于存放多个页面可能共用的资源，避免重复引入。
- 页面内如需引用外部资源，请使用相对路径指向 `assets/`。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 每个页面应自包含，或仅引用 `assets/` 下的资源。
- 页面内容由提交者自行负责，仓库仅做存储与版本管理。
- 建议每个页面添加简短的注释说明其用途或来源。

## 维护说明

- 欢迎提交新的 HTML 页面，请将文件放入 `pages/` 目录。
- 提交前请确保页面在不同浏览器下基本可用。
- 不建议在页面中包含指向不明或可能失效的外部链接。
- 仓库维护者会定期检查页面完整性，但不对内容做额外审查。

## 其他

- 本仓库遵循开源精神，不限制使用场景，但不对使用后果负责。
- 如需引用本仓库中的页面，请保留原始出处信息（如有）。
